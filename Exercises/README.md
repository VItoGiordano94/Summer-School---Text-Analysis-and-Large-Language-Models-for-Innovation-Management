# Exercises

This folder contains the **R scripts, datasets, instructions, and supporting files** used during the practical exercises of the **Summer School on Text Analysis and Large Language Models for Innovation Management**, organised by the University of Pisa.

The exercises are designed to help participants apply the concepts and methods introduced during the lessons. They provide hands-on experience with the main stages of a text-analysis project, from importing and preparing textual data to applying analytical methods and interpreting the results.

## How to Use the Exercises

Participants should download or clone the repository and open the relevant R script in **RStudio** or another R development environment.

Before running an exercise, participants should:

1. read the instructions included in the script or in the exercise-specific README file;
2. verify that all required datasets are available;
3. install the necessary R packages;
4. set the correct working directory or use an R project;
5. run the code progressively and examine the resulting outputs;
6. complete the requested tasks and interpret the results.

Participants are encouraged to modify the code, test alternative parameters, and adapt the exercises to their own research questions and datasets.

### Project Works

The repository includes a collection of project works designed to apply Natural Language Processing (NLP) techniques to real-world innovation management problems. Each project focuses on a different analytical setting and guides participants through a complete text analysis workflow.

- **Project Work 1 – Text Pre-processing of Smartphone User Reviews**
  - **Setting:** User reviews of smartphones collected from Amazon.
  - **Objective:** Learn the fundamentals of text preprocessing by cleaning, tokenizing, lemmatizing, and exploring textual data through word frequencies and Part-of-Speech (PoS) analysis.

- **Project Work 2 – Topic Modeling on Smartphone User Reviews**
  - **Setting:** The same corpus of smartphone reviews, using the preprocessed texts from Project Work 1.
  - **Objective:** Discover the main themes discussed by users through topic modeling and interpret the latent topics emerging from customer feedback.

- **Project Work 3 – Sentiment Analysis on Smartphone User Reviews**
  - **Setting:** Smartphone reviews from Amazon.
  - **Objective:** Measure the polarity of user opinions by classifying reviews as positive, negative, or neutral, and identify the reviews expressing the strongest positive and negative sentiments.

- **Project Work 4 – Extracting Smartphone Components from User Reviews**
  - **Setting:** Smartphone reviews mentioning product features and hardware components.
  - **Objective:** Build a gazetteer of smartphone components and apply Named Entity Recognition (NER) techniques to identify the most frequently discussed product features.

- **Project Work 5 – Mining Insights from Textual Data**
  - **Setting:** Smartphone user reviews analyzed through the outputs of the previous project works.
  - **Objective:** Integrate preprocessing, topic modeling, sentiment analysis, and entity extraction to answer innovation-related research questions, such as identifying the most appreciated smartphone features, uncovering latent customer needs, or suggesting functionalities for future product development.
